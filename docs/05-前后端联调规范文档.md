# 电影点评系统前后端联调规范文档

## 1. 文档目标

本文档用于统一前后端开发和联调规则，减少接口字段不一致、状态不一致、权限不一致和错误处理不一致的问题。

## 2. 环境约定

| 环境 | 用途 | 示例 |
| --- | --- | --- |
| local | 本地开发 | http://localhost:8080 |
| dev | 小组联调环境 | http://dev.example.com |
| test | 测试环境 | http://test.example.com |

前端通过环境变量维护后端地址，例如：

```text
VITE_API_BASE_URL=http://localhost:8080/api
```

## 4. 接口统一规范

### 4.1 响应结构

所有接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

分页接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],
    "page": 1,
    "pageSize": 10,
    "total": 100
  }
}
```

### 4.2 字段命名

前后端 JSON 字段统一使用 camelCase，例如：

```json
{
  "movieId": 1,
  "totalScore": 9.0,
  "createdAt": "2026-07-04 16:00:00"
}
```

数据库字段可以使用 snake_case，但接口返回必须统一为 camelCase。

### 4.3 时间格式

统一使用：

```text
yyyy-MM-dd HH:mm:ss
```

示例：

```text
2026-07-04 16:00:00
```

### 4.4 金额格式

周边商品价格使用数字类型，保留两位小数：

```json
{
  "price": 199.00
}
```

## 5. 登录与 Token 联调

### 5.1 登录返回

登录成功后返回 Token 和用户信息：

```json
{
  "token": "jwt-token",
  "user": {
    "id": 1,
    "username": "user001",
    "nickname": "电影爱好者",
    "roles": ["USER"]
  }
}
```

### 5.2 前端存储

前端保存 Token 后，每次请求登录后接口时添加：

```text
Authorization: Bearer <token>
```

### 5.3 Token 失效

当后端返回 401 时，前端应：

- 清除本地 Token。
- 跳转登录页。
- 提示登录已失效。

## 6. 权限联调规则

### 6.1 后端返回

无权限统一返回：

```json
{
  "code": 403,
  "message": "无权限操作",
  "data": null
}
```

### 6.2 前端处理

- 页面级无权限：跳转无权限页或显示无权限提示。
- 按钮级无权限：隐藏按钮。
- 操作级无权限：弹窗提示。

### 6.3 必测场景

- 游客访问评分接口，应返回 401。
- 普通用户访问管理员后台接口，应返回 403。
- 电影商修改其他电影商投稿，应返回 403。
- 管理员访问超级管理员权限接口，应返回 403。

## 7. 审核状态联调规则

### 7.1 状态枚举

前后端统一使用：

```text
DRAFT, PENDING, APPROVED, ONLINE, REJECTED, OFFLINE, DELETED
```

前端展示文案：

| 后端状态 | 前端文案 |
| --- | --- |
| DRAFT | 草稿 |
| PENDING | 待审核 |
| APPROVED | 已通过 |
| ONLINE | 已上线 |
| REJECTED | 已驳回 |
| OFFLINE | 已下架 |
| DELETED | 已删除 |

### 7.2 状态流转

```text
DRAFT -> PENDING -> APPROVED -> ONLINE
PENDING -> REJECTED -> PENDING
ONLINE -> OFFLINE
APPROVED -> OFFLINE
任意后台可删除状态 -> DELETED
```

### 7.3 驳回原因

驳回接口必须传 rejectReason。前端应在电影商后台或个人长评列表中展示驳回原因。

## 8. 分页、搜索与筛选

### 8.1 分页参数

所有列表接口统一：

```text
page=1&pageSize=10
```

### 8.2 排序参数

常用排序：

| sort | 说明 |
| --- | --- |
| latest | 最新 |
| hot | 热门 |
| score | 评分 |
| views | 浏览量 |

### 8.3 空结果处理

后端返回空数组：

```json
{
  "list": [],
  "page": 1,
  "pageSize": 10,
  "total": 0
}
```

前端显示空状态提示。

## 9. 文件上传联调

### 9.1 上传接口

```text
POST /api/uploads/images
```

请求格式：

```text
multipart/form-data
```

返回：

```json
{
  "url": "https://example.com/uploads/image.jpg"
}
```

### 9.2 限制

- 文件类型：jpg、jpeg、png、webp。
- 单文件大小不超过 5MB。
- 后端需要校验 MIME 类型和扩展名。

## 10. 模块联调顺序

建议按以下顺序联调：

1. 用户注册、登录、Token、当前用户信息。
2. 角色权限和后台入口。
3. 电影列表、电影详情。
4. 评分、短评。
5. 通用审核流程。
6. 长评发布、审核、详情、回复。
7. 资讯列表、详情、首页推荐。
8. 视频和周边列表、详情、跳转统计。
9. 电影商认证和投稿。
10. 管理员后台审核、推荐位、举报处理。
11. AI 助手推荐跳转。

## 11. Mock 数据约定

接口未完成时，前端可使用 Mock 数据，但字段必须与接口文档保持一致。

示例电影卡片：

```json
{
  "id": 1,
  "title": "星际穿越",
  "posterUrl": "https://example.com/poster.jpg",
  "genres": ["科幻", "剧情"],
  "releaseDate": "2014-11-12",
  "avgTotalScore": 9.4
}
```

示例审核内容：

```json
{
  "targetType": "NEWS",

  "submitterName": "某电影官方",
  "status": "PENDING",

}
```

## 12. 联调验收清单

### 12.1 用户与权限

- 注册成功后能登录。
- 登录后能获取当前用户信息。
- 不同角色看到不同入口。
- 无权限接口正确返回 403。

### 12.2 电影、评分、短评

- 电影搜索参数生效。
- 电影详情包含评分、短评、资讯、视频和周边。
- 总评分必填校验生效。
- 重复评分更新原评分。
- 短评发布后立即展示。

### 12.3 长评与审核

- 长评提交后状态为 PENDING。
- 管理员通过后可上线。
- 上线长评前台可见。
- 驳回长评展示驳回原因。
- 回复、点赞、收藏、举报可用。

### 12.4 资讯、视频、周边

- 资讯列表和详情可打开。
- 首页轮播可以跳转资讯详情。
- 视频点击跳转第三方链接并记录点击。
- 周边商品可以筛选并跳转外部购买链接。

### 12.5 后台

- 管理员可查看待审核列表。
- 审核动作会生成审核记录。
- 推荐位配置后首页展示变化。
- 举报处理后内容状态变化。

## 13. 常见问题处理

| 问题 | 处理方式 |
| --- | --- |
| 字段名对不上 | 以 API 文档为准，统一 camelCase |
| 状态显示不一致 | 以后端枚举为准，前端统一映射文案 |
| 接口 401 | 检查 Token 是否缺失或过期 |
| 接口 403 | 检查角色、权限和数据归属 |
| 分页总数不对 | 后端检查 total 是否按筛选条件统计 |
| 图片不显示 | 检查上传返回 URL 是否可访问 |
