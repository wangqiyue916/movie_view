# 电影点评系统后端模块详细设计与类设计文档

## 1. 文档目标

本文档用于指导后端开发人员理解系统后端模块拆分、类职责、核心方法、对象分层和模块调用关系。本文档不要求列出每一行代码实现，而是约定开发时必须遵循的类结构和关键方法。

后端建议技术栈：

- Java 17
- Spring Boot
- Spring Web
- MyBatis / MyBatis-Plus
- MySQL 8.x
- JWT

## 2. 后端总体分层

后端采用经典分层结构：

```text
Controller -> Service -> Mapper -> Database
```

各层职责：

| 层级 | 职责 |
| --- | --- |
| Controller | 接收请求、参数校验、调用 Service、返回统一响应 |
| Service | 业务逻辑、事务控制、权限校验、状态流转 |
| Mapper | 数据库读写 |
| Entity | 数据库表映射对象 |
| DTO / Request | 接收前端请求 |
| VO / Response | 返回前端展示 |
| Enum | 状态、角色、类型等枚举 |

要求：

- Controller 不直接调用 Mapper。
- Entity 不直接返回给前端。
- Service 层负责事务和业务规则。
- 通用审核状态必须通过 AuditService 流转。

## 4. 通用基础类设计

### 4.1 统一响应类

类名：

```text
ApiResponse<T>
```

职责：

- 统一封装接口返回格式。
- 支持成功响应和失败响应。

核心字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| code | Integer | 状态码 |
| message | String | 提示信息 |
| data | T | 返回数据 |

核心方法：

```text
success(data)
success()
error(code, message)
```

### 4.2 分页响应类

类名：

```text
PageResult<T>
```

核心字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| list | List<T> | 当前页数据 |
| page | Integer | 当前页 |
| pageSize | Integer | 每页数量 |
| total | Long | 总数 |

### 4.3 业务异常类

类名：

```text
BusinessException
```

核心字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| code | Integer | 错误码 |
| message | String | 错误信息 |

### 4.4 全局异常处理器

类名：

```text
GlobalExceptionHandler
```

职责：

- 捕获 BusinessException。
- 捕获参数校验异常。
- 捕获未登录、无权限异常。
- 捕获系统异常。
- 返回统一 ApiResponse。

### 4.5 登录上下文

类名：

```text
LoginUserContext
```

职责：

- 获取当前登录用户 ID。
- 获取当前用户角色。
- 判断当前用户是否为管理员、超级管理员、电影官方用户。

核心方法：

```text
getCurrentUserId()
getCurrentUser()
hasRole(roleCode)
hasPermission(permissionCode)
```

### 15.4 审核状态流转

```text
DRAFT -> PENDING
PENDING -> APPROVED
PENDING -> REJECTED
REJECTED -> PENDING
APPROVED -> ONLINE
ONLINE -> OFFLINE
OFFLINE -> ONLINE
任意后台允许删除状态 -> DELETED
```

事务要求：

- 修改业务表状态和写入 audit_records 必须在同一事务中。
- 驳回必须填写 rejectReason。

## 22. 后端开发顺序建议

1. 创建项目基础结构。
2. 实现 common 包，包括统一响应、异常、枚举。
3. 实现数据库连接和基础配置。
4. 实现用户注册、登录、Token。
5. 实现角色权限和拦截器。
6. 实现电影基础查询。
7. 实现评分和短评。
8. 实现通用审核服务。
9. 实现长评、资讯、视频、周边投稿并接入审核。
10. 实现管理员后台和电影商后台。
11. 实现首页聚合。
12. 实现 AI 助手。

## 23. 后端交付检查清单

- Controller、Service、Mapper 分层清晰。
- Entity 不直接返回给前端。
- DTO 和 VO 命名符合规范。
- 统一使用 ApiResponse。
- 列表接口使用 PageResult。
- 需要登录的接口校验 Token。
- 后台接口校验角色和权限。
- 数据归属已校验。
- 软删除查询已过滤。
- 多表更新已加事务。
- 审核状态通过 AuditService 流转。
- 异常统一返回错误码。
