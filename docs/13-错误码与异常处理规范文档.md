# 电影点评系统错误码与异常处理规范文档

## 1. 文档目标

本文档用于统一系统错误码、异常分类、后端异常处理、前端错误提示和常见业务异常，保证前后端联调时错误响应一致。

## 2. 统一响应格式

所有接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

错误返回：

```json
{
  "code": 400,
  "message": "请求参数错误",
  "data": null
}
```

## 3. 通用错误码

| 错误码 | 名称 | 说明 |
| --- | --- | --- |
| 0 | SUCCESS | 成功 |
| 400 | BAD_REQUEST | 请求参数错误 |
| 401 | UNAUTHORIZED | 未登录或 Token 失效 |
| 403 | FORBIDDEN | 无权限 |
| 404 | NOT_FOUND | 资源不存在 |
| 409 | CONFLICT | 状态冲突或重复提交 |
| 429 | TOO_MANY_REQUESTS | 请求过于频繁 |
| 500 | INTERNAL_ERROR | 服务器内部错误 |

## 5. 后端异常分类

建议后端统一定义以下异常：

| 异常 | 用途 |
| --- | --- |
| BusinessException | 业务异常 |
| UnauthorizedException | 未登录异常 |
| ForbiddenException | 无权限异常 |
| ResourceNotFoundException | 资源不存在 |
| StateConflictException | 状态冲突 |
| ValidationException | 参数校验异常 |

## 6. 异常处理规则

### 6.1 Controller 层

Controller 不直接拼接错误响应。

错误统一交给全局异常处理器处理。

### 6.2 Service 层

Service 发现业务错误时抛出业务异常。

示例：

```text
throw new BusinessException(5003, "驳回原因不能为空");
```

### 6.3 全局异常处理

后端应提供全局异常处理器，统一转换为响应结构：

```json
{
  "code": 5003,
  "message": "驳回原因不能为空",
  "data": null
}
```

## 7. 前端错误处理规则

### 7.1 统一拦截

前端 Axios 响应拦截器统一处理：

- 401：清除 Token 并跳转登录页。
- 403：提示无权限。
- 500：提示服务器异常。
- 其他业务错误：展示后端 message。

### 7.2 页面级处理

| 场景 | 处理 |
| --- | --- |
| 列表加载失败 | 显示错误提示并保留重试按钮 |
| 详情不存在 | 显示资源不存在提示 |
| 表单提交失败 | 保留用户输入并显示错误原因 |
| 登录失效 | 跳转登录页 |
| 无权限页面 | 跳转无权限页 |

## 8. 参数校验异常

后端应校验：

- 必填参数。
- 字符串长度。
- 数字范围。
- 枚举值是否合法。
- URL 格式。
- 文件类型和大小。

参数错误统一返回：

```json
{
  "code": 400,
  "message": "请求参数错误",
  "data": null
}
```

如能明确字段，可返回更具体提示：

```json
{
  "code": 3003,
  "message": "总评分不能为空",
  "data": null
}
```

## 9. 状态冲突异常

以下场景返回 409 或对应业务错误码：

- 重复点赞。
- 重复收藏。
- 重复上线。
- 重复下架。
- 审核状态不允许流转。
- 修改已删除内容。
- 电影商修改他人投稿。

示例：

```json
{
  "code": 5002,
  "message": "当前状态不允许执行该操作",
  "data": null
}
```

## 10. 日志记录规范

### 10.1 需要记录日志的场景

- 登录失败。
- 权限拒绝。
- 审核操作失败。
- 数据库异常。
- 文件上传失败。
- 第三方链接跳转统计失败。

### 10.2 日志注意事项

日志中禁止记录：

- 明文密码。
- Token 完整内容。
- 数据库密码。
- 用户敏感证件完整信息。

## 12. 前后端联调检查清单

- 所有接口是否使用统一响应结构。
- 401 是否能跳转登录。
- 403 是否能提示无权限。
- 参数错误是否能展示明确提示。
- 审核状态错误是否返回状态冲突。
- 重复点赞、收藏、评分是否处理。
- 后端是否避免返回堆栈信息。
- 日志中是否避免输出敏感信息。
