# 电影点评系统测试与验收规范文档

## 1. 文档目标

本文档用于规定电影点评系统各模块的测试范围、测试方法、验收标准和交付要求，确保项目功能完整、权限正确、数据一致、联调稳定。

## 2. 测试类型

| 测试类型 | 说明 |
| --- | --- |
| 功能测试 | 验证每个功能是否按需求正常运行 |
| 接口测试 | 验证 API 入参、返回、错误码和权限 |
| 权限测试 | 验证不同角色访问范围 |
| 数据库测试 | 验证数据写入、状态流转和软删除 |
| 联调测试 | 验证前后端交互是否一致 |
| 异常测试 | 验证参数错误、未登录、无权限、状态冲突等场景 |
| 演示验收 | 验证最终答辩演示流程 |

## 3. 测试账号建议

| 角色 | 用户名示例 | 用途 |
| --- | --- | --- |
| 普通用户 | user_test | 评分、短评、长评、回复 |
| 电影官方用户 | official_test | 认证申请、投稿 |
| 管理员 | admin_test | 审核、内容管理、举报处理 |
| 超级管理员 | super_admin | 管理员账号和权限 |

密码由项目负责人统一设置，不写入公开文档。

## 4. 通用验收标准

每个模块验收时必须满足：

- 页面可以正常打开。
- 接口请求成功。
- 必填项校验有效。
- 列表支持分页。
- 数据能正确写入数据库。
- 错误情况有清晰提示。
- 无权限访问能被拦截。
- 前台公开页面只展示 ONLINE 状态内容。
- 删除操作默认软删除。

## 14. 演示验收流程

建议最终答辩演示按以下顺序：

1. 展示首页轮播、电影栏目、资讯栏目和优质长评。
2. 搜索电影并进入电影详情页。
3. 展示评分、短评、长评、资讯、视频和周边。
4. 普通用户登录并提交评分、短评、长评。
5. 管理员登录后台审核长评。
6. 长评上线后前台展示。
7. 电影商登录后台提交资讯或周边。
8. 管理员审核电影商投稿。
9. 展示权限拦截和无权限提示。
10. 展示 AI 助手推荐跳转。

## 15. 模块交付要求

每个成员交付模块时至少提供：

- 已完成页面截图。
- 接口测试截图或说明。
- 数据库数据变化说明。
- 关键功能演示步骤。
- 已知问题清单。
- 与其他模块的对接说明。
