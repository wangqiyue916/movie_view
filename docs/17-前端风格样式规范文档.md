# 电影点评系统前端风格样式规范文档

## 1. 文档目标

本文档用于统一电影点评系统前台页面的视觉风格、布局方式、组件样式和交互表现。

当前项目前台风格以首页为样板，整体采用“黑金电影感”设计：黑色作为页面主背景，暗金色作为品牌识别、标题、边框、装饰线和交互高亮色，营造影院、胶片、海报、影评社区的视觉氛围。

本规范主要适用于：

- 首页
- 电影列表页
- 电影详情页
- 电影资讯页
- 长评列表页
- 长评详情页
- 周边商城前台页
- AI 助手前台悬浮窗

后台管理页面、电影商后台页面可以保持更偏业务管理系统的简洁风格，但色彩和品牌元素应与前台保持基本一致。

## 2. 整体视觉定位

### 2.1 关键词

- 黑金
- 影院感
- 胶片感
- 高级感
- 海报陈列
- 深色沉浸
- 内容优先

### 2.2 设计原则

1. 页面背景以黑色和近黑色为主，避免大面积白底。
2. 金色只作为强调色，不大面积铺满。
3. 内容卡片保持低亮度、半透明、细边框。
4. 图片是前台页面的核心视觉资产，电影、资讯、周边都应优先使用图片承载氛围。
5. 页面要支持纵向浏览，模块之间用暗金细线或留白分隔。
6. 装饰元素必须克制，不遮挡内容，不干扰文字阅读。

## 3. 色彩规范

### 3.1 主色

| 用途 | 色值 | 说明 |
| --- | --- | --- |
| 页面主背景 | `#050505` | 前台页面最主要背景色 |
| 次级深色背景 | `#0e0c08` | 用于渐变过渡和深色层次 |
| 卡片背景 | `rgb(255 255 255 / 6%)` | 深色背景上的半透明卡片 |
| 金色主强调 | `#d6b05f` | 标题线、分页点、边框高亮 |
| 金色文字 | `#e8c16d` | 模块标题、品牌文字、重点信息 |
| 暖白标题 | `#fff8e6` | 大标题、卡片标题 |
| 正文浅金灰 | `#d8c69b` | 卡片下方说明文字 |
| 次级正文 | `#b9ab90` | 描述、摘要、辅助信息 |
| 元信息文字 | `#c6b78f` | 作者、日期、统计数据 |

### 3.2 使用规则

- 页面背景必须使用黑色或近黑色。
- 金色用于强调，不用于大面积文本段落。
- 普通正文不要使用纯白，建议使用 `#b9ab90` 或 `#d8c69b`。
- 大标题可以使用暖白色 `#fff8e6`，并配合轻微金色阴影。
- 边框优先使用透明金色，例如 `rgb(214 176 95 / 22%)`。

## 4. 字体规范

### 4.1 字体族

普通内容字体：

```css
font-family:
  Inter,
  "PingFang SC",
  "Microsoft YaHei",
  Arial,
  sans-serif;
```

品牌、模块标题、主视觉标题：

```css
font-family: "Noto Serif SC", "Songti SC", SimSun, serif;
```

### 4.2 字号建议

| 元素 | 建议字号 |
| --- | --- |
| 顶部品牌名 | `20px` |
| 首页轮播主标题 | `clamp(38px, 5vw, 68px)` |
| 首页模块标题 | `clamp(22px, 2.8vw, 34px)` |
| 分区标题 | `18px` |
| 卡片标题 | `17px - 20px` |
| 正文摘要 | `13px - 15px` |
| 元信息 | `12px - 13px` |
| 评分数字 | `17px - 30px`，根据卡片大小决定 |

### 4.3 字体使用规则

- 不使用负 letter-spacing。
- 不使用随视口无限放大的字体。
- 模块标题要有印刷感，但不能过大压迫内容。
- 卡片内标题必须小于页面模块标题。
- 长文本摘要最多展示 2 到 3 行，避免撑高卡片。

## 5. 页面背景与装饰规范

### 5.1 背景层

前台页面背景建议由以下层组成：

- 黑色底色
- 暗金径向光晕
- 少量金色颗粒点
- 极淡竖向纹理
- 左右胶片装饰带

示例：

```css
background:
  radial-gradient(circle at 8% 18%, rgb(214 176 95 / 14%), transparent 24%),
  radial-gradient(circle at 92% 26%, rgb(214 176 95 / 12%), transparent 24%),
  linear-gradient(180deg, #050505 0%, #0e0c08 46%, #050505 100%);
```

### 5.2 两侧装饰

两侧黑色空白区域推荐使用：

- 竖向半透明胶片带
- 暗金细线
- 少量金色颗粒
- 低透明度光晕

要求：

- 装饰层必须放在内容层下方。
- 不得遮挡轮播、卡片、文字。
- 透明度建议控制在 `0.2 - 0.5`。
- 装饰宽度建议 `80px - 120px`。

## 6. 首页轮播规范

### 6.1 轮播结构

首页顶部使用 4 张图的矩形轮播展示。

轮播内容包括：

- 当前主展示图
- 左侧上一张图
- 右侧下一张图
- 隐藏的后备图

### 6.2 轮播尺寸

桌面端建议：

```css
width: min(920px, 76vw);
height: 340px;
```

中等屏幕：

```css
width: min(760px, 88vw);
height: 292px;
```

移动端：

```css
width: 90vw;
height: 250px;
```

### 6.3 轮播交互

- 自动轮换时间：`10s`
- 动画时长：`900ms`
- 主图向左后退，右侧图进入主展示位。
- 左右图片可以被主图遮挡一部分。
- 左右图片需要有透视、缩小、暗化和淡化效果。
- 移动端可以隐藏左右侧图，仅保留主图轮换。

### 6.4 轮播视觉

主图：

- 边框：暗金细边框
- 圆角：`8px`
- 阴影：黑色大阴影
- 文案居中
- 图片上叠加黑色蒙层，保证文字可读

分页点：

- 使用横向短线，不使用大圆点
- 当前项使用金色高亮

## 7. 内容模块布局规范

### 7.1 页面模块顺序

首页当前标准顺序为：

1. 顶部热门轮播
2. 热门电影
3. 电影资讯
4. 优质长评

其他前台页面可以根据业务调整顺序，但模块之间应保持纵向浏览逻辑。

### 7.2 模块标题

模块标题样式：

- 左侧使用一条短金线。
- 标题使用金色印刷体。
- 字号控制在 `22px - 34px`。
- 模块之间用暗金顶部分割线区分。

示例：

```css
.section-heading h2 {
  color: #e8c16d;
  font-family: "Noto Serif SC", "Songti SC", SimSun, serif;
  font-size: clamp(22px, 2.8vw, 34px);
}
```

## 8. 热门电影板块规范

### 8.1 分区结构

热门电影板块分为 5 个子板块：

- 近期热门
- 高分推荐
- 华语新片
- 欧美新片
- 日韩新片

每个子板块使用横向海报列表。

### 8.2 海报卡片

海报卡片包含：

- 电影海报
- 右下角评分
- 海报下方文字：`上映年份/电影名`

推荐尺寸：

```css
grid-auto-columns: 138px;
poster-height: 196px;
```

### 8.3 横向滚动

当电影数量超过屏幕宽度时，使用横向滚动，不挤压海报。

要求：

- 每个海报宽度固定。
- 海报之间间距建议 `14px`。
- 不允许海报被拉伸变形。
- 评分固定在海报右下角。

## 9. 电影资讯板块规范

电影资讯卡片应比电影海报板块更紧凑。

卡片结构：

- 左侧资讯封面
- 右侧分类、标题、摘要

推荐桌面端布局：

```css
grid-template-columns: 132px 1fr;
min-height: 128px;
```

规则：

- 标题最多 2 行。
- 摘要控制在 2 到 3 行。
- 分类使用金色小字。
- 卡片背景使用深色半透明。

## 10. 优质长评板块规范

长评以“一条长评占一行”的方式纵向排列。

每条长评展示：

- 长评标题
- 开头部分内容
- 作者
- 发布日期
- 点赞量
- 评论量

规则：

- 长评摘要最多 2 行。
- 元信息放在卡片底部。
- 不使用大面积引用符装饰，避免干扰阅读。
- 卡片高度保持紧凑，建议 `128px` 左右。

## 11. 卡片与边框规范

### 11.1 通用卡片

```css
border: 1px solid rgb(214 176 95 / 22%);
border-radius: 8px;
background: linear-gradient(180deg, rgb(255 255 255 / 6%), rgb(255 255 255 / 2%));
box-shadow: 0 12px 28px rgb(0 0 0 / 34%);
```

### 11.2 圆角

- 普通卡片：`8px`
- 海报图片：`6px`
- 顶部轮播：`8px`
- 按钮和分页短线可使用较大圆角

### 11.3 悬停效果

海报悬停：

- 图片轻微放大：`scale(1.04)`
- 亮度略微提高
- 不做过度夸张动画

普通卡片悬停：

- 可轻微提高边框金色透明度
- 可轻微上浮 `2px - 4px`

## 12. 导航栏规范

首页导航使用黑金风格：

- 背景：`rgb(5 5 5 / 94%)`
- 底部分割线：透明金色
- 品牌名：金色印刷体
- 菜单文字：浅金灰
- hover：亮金色

其他普通页面可以继续使用较简洁布局，但如需要统一前台风格，建议逐步迁移到黑金导航。

## 13. 响应式规范

### 13.1 桌面端

- 页面最大内容宽度可控制在 `1280px` 左右。
- 顶部轮播居中，左右露出上一张和下一张。
- 热门电影横向排列。
- 资讯可以三列展示。
- 长评纵向列表展示。

### 13.2 平板端

- 顶部轮播宽度变为 `88vw`。
- 资讯改为单列或双列。
- 海报列表继续横向滚动。

### 13.3 手机端

- 轮播只保留主图，隐藏左右侧图。
- 资讯卡片改为上下结构。
- 海报横向滚动保留。
- 不允许文字溢出容器。

## 14. 图片资源规范

### 14.1 图片类型

前台页面应优先使用：

- 电影海报
- 影院场景
- 剧照
- 资讯封面
- 周边商品图
- 解读视频封面

### 14.2 图片要求

- 图片必须清晰，不使用模糊低质量图片。
- 海报比例保持竖版，不拉伸。
- 资讯封面使用横图或可裁剪横图。
- 顶部轮播图应有足够暗部，方便叠加文字。
- 外链图片要注意稳定性，避免使用容易防盗链的平台图片。

## 15. 交互规范

### 15.1 轮播

- 自动轮换：10 秒。
- 支持点击分页短线切换。
- 动画平滑，不突然闪烁。
- 当前展示内容必须始终居中。

### 15.2 横向海报列表

- 支持鼠标滚轮或触控横向浏览。
- 不做自动滚动，避免干扰用户阅读。
- hover 只做轻微反馈。

### 15.3 卡片点击

后续接入真实路由时：

- 点击电影海报进入电影详情页。
- 点击资讯卡片进入资讯详情页。
- 点击长评卡片进入长评详情页。

## 16. 禁止事项

前台页面禁止出现以下情况：

- 大面积白色背景。
- 大量高饱和纯黄色。
- 过多渐变色堆叠导致页面发脏。
- 卡片套卡片。
- 文字压在复杂图片上但没有蒙层。
- 海报被横向或纵向拉伸变形。
- 首页模块标题过大，抢占内容视线。
- 装饰元素遮挡内容。
- 同一页面出现多套完全不同的视觉风格。
- 使用纯说明文字代替真实内容展示。

## 17. 开发落地建议

### 17.1 样式组织

首页当前样式可作为前台风格样板。后续建议逐步抽取：

```text
src/styles/theme.css
src/components/common/SectionHeading.vue
src/components/movie/PosterCard.vue
src/components/news/NewsCompactCard.vue
src/components/review/ReviewListItem.vue
```

### 17.2 CSS 变量建议

后续可以在全局样式中定义：

```css
:root {
  --color-bg-black: #050505;
  --color-bg-deep: #0e0c08;
  --color-gold: #d6b05f;
  --color-gold-text: #e8c16d;
  --color-title-warm: #fff8e6;
  --color-text-main: #d8c69b;
  --color-text-muted: #b9ab90;
  --color-border-gold: rgb(214 176 95 / 22%);
}
```

### 17.3 组件复用

开发新页面时优先复用以下模式：

- 页面背景：黑色 + 暗金装饰。
- 模块标题：金色印刷体 + 左侧短线。
- 电影内容：海报卡片。
- 资讯内容：紧凑图文卡片。
- 长评内容：纵向列表项。
- 统计信息：小号金灰文字。

## 18. 与当前首页样板对应关系

当前样板文件：

```text
frontend/src/views/home/HomePage.vue
frontend/src/layouts/MainLayout.vue
```

首页样板已包含：

- 黑金背景
- 两侧胶片装饰
- 顶部 3D 轮播
- 热门电影五分区横向海报列表
- 紧凑资讯卡片
- 纵向长评列表
- 首页专用黑金导航

后续其他前台页面应以该首页为视觉基准进行扩展。
