# 电影点评系统部署与运行说明文档

## 1. 文档目标

本文档用于说明电影点评系统在本地开发、联调环境和演示环境中的启动方式、数据库配置、前端运行、后端运行和常见问题处理。

## 2. 环境要求

### 2.1 本地开发环境

| 环境 | 建议版本 |
| --- | --- |
| JDK | 17 |
| Maven | 3.8+ |
| Node.js | 18+ |
| npm | 9+ |
| MySQL 客户端 | 8.x |
| Git | 2.x |

### 2.2 数据库环境

项目使用阿里云云数据库 RDS MySQL 版。

| 配置项 | 值 |
| --- | --- |
| 连接名称 | movie |
| 服务版本 | 8.0.18-dms-proxy-2022 |
| 主机 | dpshmy-ugp2kxl2hqqylbtj-pub.proxy.dms.aliyuncs.com |
| 端口 | 3306 |
| 用户名 | nhH6IpERCnuCcjhV6h0okozZ |
| 数据库名 | movie |
| 密码 | 通过环境变量 DB_PASSWORD 配置，不写入文档 |
| 字符集 | 自动，建议 utf8mb4 |

## 3. 敏感配置说明

数据库密码禁止写入：

- Git 仓库。
- Markdown 文档。
- 前端代码。
- 后端源码。
- 截图或公开报告。

统一通过环境变量配置：

```text
DB_PASSWORD=<数据库密码>
```

本地可使用 `.env` 或 IDE 运行配置，服务器可使用系统环境变量或部署平台的密钥配置。

## 4. 数据库初始化

### 4.1 创建数据库

如果数据库尚未创建，可执行：

```sql
CREATE DATABASE movie DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

如果阿里云 RDS 已创建数据库 `movie`，则无需重复创建。

### 4.2 导入表结构

建议将建表脚本放在：

```text
database/schema.sql
```

导入方式：

```bash
mysql -h dpshmy-ugp2kxl2hqqylbtj-pub.proxy.dms.aliyuncs.com -P 3306 -u nhH6IpERCnuCcjhV6h0okozZ -p movie < database/schema.sql
```

执行后输入数据库密码。

### 4.3 导入初始化数据

建议将初始化数据放在：

```text
database/init-data.sql
```

导入方式：

```bash
mysql -h dpshmy-ugp2kxl2hqqylbtj-pub.proxy.dms.aliyuncs.com -P 3306 -u nhH6IpERCnuCcjhV6h0okozZ -p movie < database/init-data.sql
```

## 5. 后端运行

### 5.1 后端配置

后端配置文件建议使用环境变量：

```yaml
server:
  port: 8080

spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://${DB_HOST}:${DB_PORT}/${DB_NAME}?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
```

本项目云数据库环境变量：

```text
DB_HOST=dpshmy-ugp2kxl2hqqylbtj-pub.proxy.dms.aliyuncs.com
DB_PORT=3306
DB_NAME=movie
DB_USERNAME=nhH6IpERCnuCcjhV6h0okozZ
DB_PASSWORD=<数据库密码>
```

### 5.2 Windows PowerShell 设置环境变量

```powershell
$env:DB_HOST="dpshmy-ugp2kxl2hqqylbtj-pub.proxy.dms.aliyuncs.com"
$env:DB_PORT="3306"
$env:DB_NAME="movie"
$env:DB_USERNAME="nhH6IpERCnuCcjhV6h0okozZ"
$env:DB_PASSWORD="<数据库密码>"
```

### 5.3 启动后端

进入后端目录：

```bash
cd movie-server
```

运行：

```bash
mvn spring-boot:run
```

后端默认访问地址：

```text
http://localhost:8080
```

API 基础地址：

```text
http://localhost:8080/api
```

## 6. 前端运行

### 6.1 安装依赖

```bash
cd movie-web
npm install
```

### 6.2 配置前端环境变量

`.env.development` 示例：

```text
VITE_API_BASE_URL=http://localhost:8080/api
```

### 6.3 启动前端

```bash
npm run dev
```

默认访问地址：

```text
http://localhost:5173
```

## 7. 前后端联调启动顺序

建议顺序：

1. 确认数据库可连接。
2. 导入表结构和初始化数据。
3. 启动后端服务。
4. 打开后端健康检查或测试接口。
5. 启动前端服务。
6. 前端登录测试账号。
7. 测试电影列表、详情、评分、审核等核心流程。

## 8. 演示环境准备

答辩演示前检查：

- 数据库连接可用。
- 后端启动成功。
- 前端启动成功。
- 测试账号可登录。
- 首页有展示数据。
- 至少有 3 部电影数据。
- 至少有 2 条资讯。
- 至少有 1 条待审核长评。
- 至少有 1 条电影商投稿。
- 管理员账号可审核内容。

## 10. 常见问题

### 10.1 数据库连接失败

检查：

- 主机地址是否正确。
- 端口是否为 3306。
- 用户名是否正确。
- 密码是否正确。
- 阿里云 RDS 是否允许当前网络访问。
- 数据库名是否为 movie。

### 10.2 后端启动失败

检查：

- JDK 版本。
- Maven 依赖是否下载完成。
- application.yml 是否配置正确。
- DB_PASSWORD 是否已设置。
- 数据库是否可连接。

### 10.3 前端请求失败

检查：

- 后端是否启动。
- `VITE_API_BASE_URL` 是否正确。
- 浏览器控制台是否有跨域错误。
- Token 是否失效。

### 10.4 401 或 403

- 401 表示未登录或 Token 失效。
- 403 表示当前账号无权限。
- 检查当前登录账号角色是否正确。

### 10.5 图片不显示

检查：

- 上传接口是否成功。
- 图片 URL 是否可访问。
- 后端静态资源映射是否配置。
- 文件大小和类型是否符合要求。

## 11. 部署检查清单

- 数据库连接信息已配置。
- 密码未写入代码和文档。
- 数据库表结构已导入。
- 初始化账号已创建。
- 后端能正常启动。
- 前端能正常启动。
- 前端能成功调用后端接口。
- 登录、电影详情、评分、审核核心流程可用。
- 首页数据不为空。
