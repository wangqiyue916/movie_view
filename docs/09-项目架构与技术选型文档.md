# 电影点评系统项目架构与技术选型文档

## 1. 文档目标

本文档用于说明电影点评系统的整体架构、技术选型、项目目录、模块关系、数据库连接信息和部署方式，为开发、联调、验收和答辩提供统一依据。

## 2. 总体架构

系统采用前后端分离架构：

```text
浏览器前端
  -> 前端 Vue 应用
  -> 后端 Spring Boot API
  -> MySQL 数据库
  -> 文件存储或本地上传目录
  -> 第三方视频/电商外链
```

用户通过浏览器访问前端页面，前端通过 HTTP API 调用后端服务。后端负责认证授权、业务逻辑、审核流程、数据库读写和统计字段维护。

## 3. 技术选型

### 3.1 前端技术栈

| 技术 | 用途 |
| --- | --- |
| Vue 3 | 前端框架 |
| Vite | 前端构建工具 |
| Vue Router | 前端路由 |
| Pinia | 状态管理 |
| Element Plus | UI 组件库 |
| Axios | HTTP 请求 |
| Markdown 渲染库 | 长评 Markdown 展示 |

### 3.2 后端技术栈

| 技术 | 用途 |
| --- | --- |
| Java 17 | 后端开发语言 |
| Spring Boot | 后端应用框架 |
| Spring Web | REST API |
| Spring Security 或拦截器 | 登录认证与权限控制 |
| MyBatis / MyBatis-Plus | 数据库访问 |
| JWT | 登录 Token |
| Maven | 依赖管理 |
| Lombok | 简化实体代码 |

### 3.3 数据库与存储

| 技术 | 用途 |
| --- | --- |
| MySQL 8.x | 业务数据库 |
| 阿里云 RDS MySQL | 云数据库环境 |
| 本地上传目录或对象存储 | 图片上传 |

### 3.4 可选扩展

| 技术 | 用途 |
| --- | --- |
| Redis | 缓存首页推荐、验证码、热点数据 |
| Elasticsearch | 复杂电影搜索 |
| 阿里云 OSS | 图片对象存储 |
| 大模型 API | AI 助手智能问答 |

第一阶段可以不接入 Redis、Elasticsearch、OSS 和真实大模型，先完成核心业务闭环。

## 10. 数据流说明

### 10.1 普通用户评分

```text
前端提交评分
  -> 后端校验登录
  -> 写入或更新 movie_ratings
  -> 重新统计 movies 平均分
  -> 返回评分结果
```

### 10.2 投稿审核

```text
投稿人提交内容
  -> 内容状态变为 PENDING
  -> 管理员后台查看待审核
  -> 管理员通过/驳回/上线/下架
  -> 写入 audit_records
  -> ONLINE 内容在前台展示
```

### 10.3 首页展示

```text
前端请求首页数据
  -> 后端查询推荐位和热门数据
  -> 只返回 ONLINE 内容
  -> 前端渲染轮播、电影、资讯、长评、周边
```

## 11. 安全架构

- 使用 Token 维护登录态。
- 后端统一进行权限校验。
- 敏感配置使用环境变量。
- 密码加密存储。
- 图片上传校验类型和大小。
- 外部链接校验协议和格式。
- 数据删除默认软删除。

## 12. 第一阶段实现范围

第一阶段重点完成：

- 用户注册、登录、退出。
- 角色与权限控制。
- 电影列表、搜索、详情。
- 评分和短评。
- 长评发布、审核、详情、回复。
- 首页展示和资讯。
- 视频推荐和周边展示。
- 电影商认证和投稿。
- 管理员后台审核流程。

AI 助手第一阶段可以先实现基于关键词的站内推荐，后续再接入大模型。
