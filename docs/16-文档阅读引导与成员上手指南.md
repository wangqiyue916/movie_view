# 电影点评系统文档阅读引导与成员上手指南

## 1. 文档目标

本文档用于指导小组成员快速理解项目文档体系，明确先读哪些文档、如何通过规范文档约束开发、如何通过开发设计文档理解项目结构并开始编码。

项目文档较多，不要求每位成员一开始全部精读。建议按角色和任务分层阅读。

## 3. 新成员快速阅读路线

### 3.1 30 分钟快速了解项目

适合第一次加入项目的人。

阅读顺序：

1. `00-功能模块说明.md`
2. `06-分工与模块边界文档.md`
3. `09-项目架构与技术选型文档.md`

目标：

- 知道系统是电影点评系统。
- 知道有哪些角色和功能模块。
- 知道自己负责的模块边界。
- 知道项目使用什么技术栈。

### 3.2 1 小时进入开发状态

适合准备写代码的人。

阅读顺序：

1. `06-分工与模块边界文档.md`
2. `03-接口API规范文档.md`
3. `02-数据库设计文档.md`
4. `14-后端模块详细设计与类设计文档.md` 或 `15-前端页面与组件设计文档.md`

目标：

- 知道自己模块有哪些接口。
- 知道自己模块用哪些表。
- 知道后端类怎么建，或前端页面怎么拆。

### 3.3 开始联调前必须阅读

阅读顺序：

1. `03-接口API规范文档.md`
2. `05-前后端联调规范文档.md`
3. `13-错误码与异常处理规范文档.md`

目标：

- 统一接口路径、参数和返回结构。
- 统一分页格式。
- 统一 401、403、业务错误处理方式。

### 3.4 提交代码前必须阅读

阅读顺序：

1. `07-数据库与后端开发规范文档.md`
2. `08-前端开发规范文档.md`
3. `11-Git协作与提交规范文档.md`

目标：

- 避免命名不一致。
- 避免提交敏感配置。
- 避免破坏他人代码。

## 5. 如何通过规范文档约束开发

### 5.1 写代码前

先确认：

- 这个功能属于哪个成员。
- 用到哪些数据库表。
- 调用哪些接口。
- 是否需要登录。
- 是否需要权限。
- 是否需要审核。
- 是否需要分页。

对应阅读：

- `06-分工与模块边界文档.md`
- `02-数据库设计文档.md`
- `03-接口API规范文档.md`
- `04-权限与角色规范文档.md`

### 5.2 写后端时

必须对照：

- `07-数据库与后端开发规范文档.md`
- `14-后端模块详细设计与类设计文档.md`
- `13-错误码与异常处理规范文档.md`

检查点：

- 表字段是否使用 snake_case。
- Java 字段是否使用 camelCase。
- Entity、DTO、VO 是否分清。
- Service 是否处理事务。
- 审核状态是否通过 AuditService 修改。
- 前台是否只查询 ONLINE 内容。
- 是否返回统一 ApiResponse。

### 5.3 写前端时

必须对照：

- `08-前端开发规范文档.md`
- `15-前端页面与组件设计文档.md`
- `03-接口API规范文档.md`

检查点：

- 页面路径是否和路由设计一致。
- API 文件是否按模块拆分。
- 请求是否通过统一 request 实例。
- 是否处理 401 和 403。
- 表单是否有必填校验。
- 列表是否支持分页。
- 状态文案是否统一。

### 5.4 联调时

必须对照：

- `03-接口API规范文档.md`
- `05-前后端联调规范文档.md`
- `13-错误码与异常处理规范文档.md`

检查点：

- 接口路径是否一致。
- 请求字段是否一致。
- 返回字段是否 camelCase。
- 分页格式是否一致。
- 错误码是否一致。
- Token 是否正确携带。

## 6. 如何通过开发设计文档上手项目

### 6.1 后端成员上手方式

1. 打开 `14-后端模块详细设计与类设计文档.md`。
2. 找到自己负责模块。
3. 按文档创建 Controller、Service、Mapper、Entity、DTO、VO。
4. 对照 `03-接口API规范文档.md` 实现接口路径。
5. 对照 `02-数据库设计文档.md` 实现表映射。
6. 对照 `07-数据库与后端开发规范文档.md` 添加事务、软删除和状态校验。
7. 对照 `10-测试与验收规范文档.md` 自测。

### 6.2 前端成员上手方式

1. 打开 `15-前端页面与组件设计文档.md`。
2. 找到自己负责页面。
3. 按文档创建页面文件和组件文件。
4. 对照 `03-接口API规范文档.md` 创建 API 调用方法。
5. 对照 `08-前端开发规范文档.md` 处理路由、权限、状态和表单校验。
6. 对照 `05-前后端联调规范文档.md` 和后端联调。

## 7. 开发前置顺序

建议按以下顺序开始项目：

1. 搭建后端 Spring Boot 基础项目。
2. 搭建前端 Vue 3 + Vite 基础项目。
3. 创建 `database/schema.sql` 和 `database/init-data.sql`。
4. 后端实现 common 基础包。
5. 后端实现用户、登录、角色、权限。
6. 前端实现路由、布局、请求封装、登录页。
7. 实现电影基础数据接口和页面。
8. 各成员开始并行开发自己的模块。

## 8. 前后端框架如何搭建

### 8.1 我可以帮你搭建什么

我可以直接在当前项目目录中生成：

```text
movie-server
movie-web
database
README.md
```

后端可以包含：

- Spring Boot 基础工程。
- Maven 配置。
- application.yml。
- 统一响应 ApiResponse。
- 全局异常处理。
- 常用枚举。
- 基础包结构。
- 数据库连接配置占位。
- 示例健康检查接口。

前端可以包含：

- Vue 3 + Vite 工程。
- 路由配置。
- Pinia。
- Axios 请求封装。
- 三套基础 Layout。
- 登录页、首页、403、404 占位页面。
- API 模块目录。
- 环境变量示例。

SQL 可以包含：

- schema.sql。
- init-data.sql。
- update 目录。

### 8.2 框架搭建后的效果

搭建完成后，组员可以直接：

1. 配置数据库环境变量。
2. 启动后端。
3. 启动前端。
4. 访问首页。
5. 测试后端健康检查接口。
6. 按分工在对应模块目录下开发。

### 8.3 搭建前需要确认的内容

如果不再变更技术栈，可以默认使用：

| 部分 | 默认选择 |
| --- | --- |
| 前端 | Vue 3 + Vite + Pinia + Element Plus |
| 后端 | Spring Boot + Maven + MyBatis-Plus |
| 数据库 | MySQL 8.x |
| 登录 | JWT |
| 包名 | com.example.movie |
| 后端端口 | 8080 |
| 前端端口 | 5173 |

如果老师或小组已指定技术栈，应先以指定技术栈为准。

## 9. 每日开发建议

每天开始开发前：

1. 拉取最新代码。
2. 看 `06-分工与模块边界文档.md` 确认自己今天做什么。
3. 看自己模块对应的开发设计文档。
4. 写代码。
5. 本地自测。
6. 提交代码前看 `11-Git协作与提交规范文档.md`。

每天结束前：

1. 提交或保存当天代码。
2. 记录未完成事项。
3. 说明是否有接口或数据库变更。
4. 如有联调问题，写清楚接口、参数、返回和截图。
