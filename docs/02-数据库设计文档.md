# 电影点评系统数据库设计文档

## 1. 设计说明

本文档基于《电影点评系统完整需求说明书》设计数据库结构。数据库建议使用 MySQL 8.x。字段类型可根据实际后端框架和数据库版本微调。

设计原则：

- 用户、角色、权限独立建模。
- 审核流程抽象为通用审核表和审核记录表。
- 业务内容表保留自身状态字段，统一使用审核状态枚举。
- 重要删除操作采用逻辑删除。
- 常用统计字段可冗余保存，便于列表和首页展示。

## 14. 表关系概览

- 一个用户可以拥有多个角色。
- 一个角色可以拥有多个权限。
- 一个电影可以拥有多条评分、短评、长评、资讯关联、视频和周边商品。
- 一个长评可以拥有多条回复和多张图片。
- 点赞、收藏、举报采用 target_type + target_id 方式关联不同内容。
- 审核记录采用 target_type + target_id 方式关联不同投稿内容。
- 首页推荐位采用 target_type + target_id 方式关联电影、资讯、长评和周边商品。
