# 电影点评系统完整需求说明书

## 1. 项目概述

### 1.1 项目名称

电影点评系统。

### 1.2 项目背景

随着电影内容、影评社区和线上资讯的快速发展，用户在选择电影时不仅关注电影简介和评分，也关注影评讨论、资讯动态、解读视频和电影周边内容。本系统面向普通观影用户、平台管理员、超级管理员和电影官方用户，提供电影浏览、评分评论、长评讨论、资讯阅读、视频推荐、周边展示、官方投稿与后台审核等功能。

### 1.3 项目目标

系统目标如下：

- 为普通用户提供电影搜索、浏览、评分、短评、长评、讨论、资讯阅读、视频查看、周边浏览和 AI 推荐服务。
- 为管理员提供电影、评论、资讯、视频、周边、首页推荐位和违规内容的管理能力。
- 为超级管理员提供系统最高权限，包括账号管理、角色管理和权限管理。
- 为电影官方用户提供认证申请、官方内容投稿、审核状态查看和数据查看能力。
- 通过统一审核流程保证内容质量、内容安全和后台管理规范。

### 1.4 用户角色

| 角色 | 说明 |
| --- | --- |
| 游客 | 未登录用户，只能浏览公开内容和搜索内容，不能评分、评论、投稿。 |
| 普通用户 | 已注册登录用户，可以评分、短评、长评、回复、点赞、收藏、举报等。 |
| 电影官方用户 | 通过认证后可投稿电影信息、资讯、视频链接、周边商品等官方内容。 |
| 管理员 | 负责内容审核、内容管理、首页推荐位配置、违规内容处理等。 |
| 超级管理员 | 拥有最高权限，可以管理用户、管理员、电影官方用户和系统权限。 |

## 2. 系统范围

### 2.1 系统包含

- 用户注册、登录、退出和角色管理。
- 电影搜索、电影列表和电影详情。
- 电影基本信息展示与管理。
- 电影总评分和细分评分。
- 短评发布、展示、点赞和举报。
- 长评发布、Markdown 展示、图片插入、回复讨论、点赞、收藏和举报。
- 首页热门资讯轮播、电影资讯栏目、热门电影、高分电影、最新电影和优质长评展示。
- 电影资讯列表、详情和关联展示。
- 电影解读视频推荐。
- 电影周边商城展示和第三方电商跳转。
- AI 助手悬浮窗和问答推荐。
- 电影官方用户认证申请和内容投稿。
- 管理员后台、审核流程、审核记录、推荐位管理和违规处理。
- 权限控制。

### 2.2 系统不包含

- 不直接实现电商交易、购物车、订单、支付、发货和库存管理。
- 不直接托管解读视频，只保存并展示第三方视频链接。
- AI 助手可以先实现规则检索或接口预留，不要求第一阶段必须接入真实大模型。
- 不要求实现复杂的多级组织权限，第一阶段以角色权限和资源归属权限为主。

## 3. 总体业务流程

### 3.1 普通用户流程

1. 用户注册并登录。
2. 搜索或浏览电影。
3. 进入电影详情页查看电影信息、评分、短评、长评、资讯、视频和周边。
4. 对电影进行总评分和可选细分评分。
5. 发布短评，短评直接展示。
6. 发布长评，长评进入待审核状态。
7. 管理员审核通过并上线后，长评对外展示。
8. 用户可以对短评、长评和回复进行点赞、收藏、举报等互动。

### 3.2 电影官方用户流程

1. 用户注册并选择或申请成为电影官方用户。
2. 提交电影商认证申请。
3. 管理员或超级管理员审核认证申请。
4. 认证通过后进入电影商后台。
5. 投稿电影信息、资讯、视频链接或周边商品。
6. 投稿内容进入统一审核流程。
7. 审核通过并上线后，对应内容在前台展示。
8. 电影官方用户查看投稿状态、驳回原因和基础数据。

### 3.3 管理员流程

1. 管理员登录后台。
2. 查看待审核投稿列表。
3. 对电影信息、资讯、视频、周边、长评等内容进行审核。
4. 可执行通过、修改后通过、驳回、上线、下架、删除等操作。
5. 管理首页推荐位和热门展示内容。
6. 处理举报内容和违规用户。
7. 查看审核记录和操作记录。

## 4. 通用审核流程需求

### 4.1 适用范围

统一审核流程适用于电影信息、电影资讯、电影解读视频、电影周边商品和用户长评。

### 4.2 审核状态

| 状态 | 说明 |
| --- | --- |
| 草稿 | 投稿人保存但未提交。 |
| 待审核 | 投稿人提交后等待审核。 |
| 已通过 | 管理员审核通过，但可能尚未上线。 |
| 已上线 | 内容已对前台用户展示。 |
| 已驳回 | 管理员审核不通过，投稿人可查看原因并修改。 |
| 已下架 | 已上线内容被管理员下架。 |
| 已删除 | 内容逻辑删除。 |

### 4.3 审核规则

- 投稿人可以保存草稿。
- 投稿人可以将草稿或被驳回内容重新提交审核。
- 待审核内容只有管理员或超级管理员可以审核。
- 管理员驳回内容时必须填写驳回原因。
- 审核操作必须生成审核记录。
- 上线内容才可以在前台公开展示。
- 已下架内容不在前台展示，但后台保留记录。
- 已删除内容默认不在列表展示，必要时后台可按条件查看。

### 4.4 审核记录字段

审核记录至少包含：

- 审核记录 ID。
- 内容类型。
- 内容 ID。
- 投稿人 ID。
- 审核人 ID。
- 审核动作。
- 审核前状态。
- 审核后状态。
- 驳回原因。
- 审核备注。
- 审核时间。

## 5. 功能性需求

### 5.1 用户与角色管理

系统应支持用户注册、登录、退出、密码加密存储、账号状态管理和用户资料维护。

系统应支持普通用户、管理员、超级管理员、电影官方用户四类核心角色，并可扩展游客访问能力。

不同角色登录后展示不同入口。普通用户进入前台，管理员和超级管理员可进入管理员后台，电影官方用户可进入电影商后台。

### 5.2 权限管理

系统应根据角色控制页面访问、接口调用和数据操作。

普通用户不能访问管理员后台和电影商后台。电影官方用户不能管理其他电影商的投稿内容。普通管理员不能执行超级管理员专属操作。

无权限访问时，前端应提示无权限，后端应返回统一错误码。

### 5.3 首页展示

首页应展示热门资讯轮播栏、电影资讯栏目、热门电影、高分电影、最新电影和优质长评。

首页电影卡片只展示总评分，不展示细分评分。

首页展示内容可以由管理员手动配置，也可以按浏览量、评分、点赞数、回复数、点击量等数据自动推荐。

### 5.4 电影搜索与浏览

系统应支持按电影名称、导演、演员、类型、上映时间等条件搜索电影。

搜索结果以列表形式展示电影海报、名称、类型、上映时间和总评分等信息。

用户点击电影后进入电影详情页。

### 5.5 电影信息展示与管理

电影详情页应展示电影名称、导演、主演、类型、上映时间、简介、海报、地区、语言、片长和评分信息。

管理员可以新增、修改、删除电影信息。

电影官方用户可以投稿或补充电影信息，投稿进入统一审核流程。

电影详情页应关联展示该电影的资讯、解读视频、长评和周边商品。

### 5.6 电影评分

登录用户可以对电影评分。

总评分为必填项，剧情评分、特效画面评分、演员演技评分为选填项。

总评分提交后立即生效。系统应计算并展示总平均分和细分平均分。

首页和电影列表只展示总评分，电影详情页展示四类评分。

同一用户对同一电影只能有一条有效评分记录，再次评分时更新原记录。

### 5.7 短评

登录用户可以在电影详情页发布短评。

短评发布后直接展示。短评支持点赞和举报。

管理员可以删除违规短评或处理被举报短评。

### 5.8 长评与讨论

登录用户可以发布长评。长评支持标题、正文、图片和 Markdown 格式。

长评提交后进入待审核状态，审核通过并上线后前台展示。

长评拥有独立详情页，用户可以点赞、收藏、回复和举报。

回复区默认支持一级回复，可扩展为多级回复。

系统可以根据点赞数、浏览量、回复数、收藏数自动推荐优质长评，也支持管理员手动推荐。

### 5.9 AI 助手推荐

系统应提供悬浮窗桌宠形式的 AI 助手入口。

用户点击后弹出对话框，可以询问演员、导演、电影、资讯、周边等相关内容。

AI 助手应返回介绍内容或推荐结果。推荐结果应支持跳转到系统内详情页或外部商品链接。

第一阶段可实现基于数据库检索和关键词匹配的问答，后续可接入大模型接口。

### 5.10 电影解读视频推荐

电影详情页应展示该电影相关的解读视频。

视频信息包括标题、封面、来源平台、简介、热度信息和外部链接。

用户点击视频后跳转到第三方视频网站。

电影官方用户可以投稿视频链接，管理员审核通过并上线后展示。

### 5.11 电影资讯

系统应提供资讯列表页和资讯详情页。

资讯内容包括新片上映、电影动态、演员新闻、导演动态、幕后花絮、获奖信息、票房新闻等。

资讯可以关联电影、演员或导演。电影详情页可以展示相关资讯。

电影官方用户可以投稿资讯，管理员审核通过并上线后展示。

### 5.12 电影周边商城

系统应提供独立电影周边商城首页。

周边商品展示商品名称、商品图片、所属电影、商品类型、价格、商品简介和第三方购买链接。

商城支持按电影名称、商品类型、价格区间等条件筛选。

系统不负责交易和库存。用户点击商品后跳转到淘宝、京东等第三方电商平台。

电影官方用户可以投稿商品信息，管理员审核通过并上线后展示。

### 5.13 管理员后台

管理员后台应包含内容审核、内容管理、首页推荐位管理、举报处理、用户处理、审核记录和操作记录等功能。

管理员可以对已上线内容进行修改、隐藏、下架或删除。

管理员可以处理违规短评、长评、回复，对用户进行警告、限制发言或封禁。

超级管理员可以管理管理员账号和角色权限。

### 5.14 电影商后台

电影商后台应支持认证申请、投稿管理、审核状态查看、驳回原因查看和基础数据查看。

电影商只能管理自己提交或被授权管理的内容。

电影商投稿内容包括电影信息、电影资讯、电影解读视频链接和电影周边商品。

## 6. 非功能性需求

### 6.1 安全性

- 密码必须加密存储。
- 登录接口应返回令牌或会话标识。
- 后端必须校验角色权限，不能只依赖前端隐藏入口。
- 外部链接应校验格式，避免恶意链接。
- 图片上传应限制文件类型和大小。

### 6.2 易用性

- 前台页面应清晰展示电影、评分、评论和资讯。
- 后台列表应支持搜索、筛选、分页和状态过滤。
- 审核驳回原因应明确展示给投稿人。
- 无权限、未登录、提交失败等情况应给出清晰提示。

### 6.3 性能

- 列表接口应支持分页。
- 首页推荐数据应限制返回数量。
- 电影评分平均值可通过缓存字段或统计表优化。
- 图片资源建议使用压缩后的访问地址。

### 6.4 可维护性

- 审核流程应设计为通用模块，避免各业务重复实现。
- 角色权限应集中管理。
- 接口返回结构应统一。
- 业务状态枚举应统一维护。

## 7. 数据统计需求

系统应记录或统计：

- 电影浏览量。
- 资讯阅读量。
- 视频点击量。
- 周边商品点击量。
- 长评浏览量、点赞数、收藏数、回复数。
- 短评点赞数和举报数。
- 用户违规次数。

## 8. 验收标准

- 用户可以完成注册、登录、退出。
- 不同角色登录后看到正确入口。
- 普通用户可以搜索电影、进入详情、评分、短评、提交长评。
- 长评提交后进入待审核，管理员通过并上线后前台可见。
- 电影官方用户可以提交认证申请，认证通过后可投稿官方内容。
- 电影、资讯、视频、周边投稿均复用统一审核状态。
- 管理员可以审核、驳回、上线、下架内容，并查看审核记录。
- 首页能展示轮播资讯、电影栏目、资讯栏目和优质长评。
- 电影详情页能展示基本信息、评分、短评、长评入口、资讯、视频和周边。
- 无权限访问后台或接口时系统能正确拦截。
