# 电影点评系统前端页面与组件设计文档

## 1. 文档目标

本文档用于指导前端开发人员理解系统页面结构、路由设计、组件拆分、页面调用接口、权限入口和模块协作方式。开发时应结合《03-接口API规范文档》和《08-前端开发规范文档》使用。

前端建议技术栈：

- Vue 3
- Vite
- Vue Router
- Pinia
- Element Plus
- Axios

## 2. 前端总体结构

```text
frontend
  └── src
      ├── api
      ├── assets
      ├── components
      ├── constants
      ├── layouts
      ├── router
      ├── stores
      ├── styles
      ├── utils
      └── views
```

## 6. Store 设计

### 6.1 userStore

职责：

- 保存 Token。
- 保存当前用户信息。
- 保存角色和权限。
- 登录后写入用户状态。
- 退出时清空用户状态。

核心状态：

```text
token
userInfo
roles
permissions
isLogin
```

核心方法：

```text
login()
logout()
fetchCurrentUser()
hasRole(role)
hasPermission(permission)
```

### 6.2 appStore

职责：

- 保存全局 loading。
- 保存侧边栏折叠状态。
- 保存主题或布局状态。

### 6.3 aiStore

职责：

- 保存 AI 对话窗口开关状态。
- 保存当前会话 ID。
- 保存消息列表。

## 11. 权限控制设计

### 11.1 路由守卫

路由进入前检查：

- 是否需要登录。
- 是否拥有角色。
- 是否拥有权限。

处理规则：

- 未登录访问登录后页面，跳转登录页。
- 无权限访问后台页面，跳转 `/403`。
- 已登录用户访问登录页，可跳转首页。

### 11.2 菜单显示

根据角色展示菜单：

- USER：前台用户菜单。
- OFFICIAL：电影商后台入口。
- ADMIN：管理员后台入口。
- SUPER_ADMIN：全部后台入口。

### 11.3 按钮显示

按钮按权限显示，例如：

- 审核通过。
- 驳回。
- 上线。
- 下架。
- 删除。
- 封禁用户。

后端仍必须做最终权限校验。

## 12. 表单设计规范

### 12.1 通用表单状态

每个表单至少包含：

- 初始状态。
- 编辑状态。
- 提交中状态。
- 提交成功状态。
- 提交失败状态。

### 12.2 必填校验

关键必填项：

- 登录：用户名、密码。
- 注册：用户名、密码、昵称。
- 评分：总评分。
- 短评：内容。
- 长评：电影、标题、正文。
- 资讯：标题、正文。
- 视频：电影、标题、平台、外链。
- 周边：电影、商品名、图片、类型、外链。
- 驳回：驳回原因。

## 13. 页面状态设计

每个列表页需要处理：

- loading。
- 空数据。
- 加载失败。
- 分页切换。
- 搜索条件变化。

每个详情页需要处理：

- loading。
- 内容不存在。
- 无权限。
- 数据加载失败。

每个提交按钮需要处理：

- 禁止重复提交。
- 提交中 loading。
- 成功提示。
- 失败提示。

## 14. 前端开发顺序建议

1. 搭建 Vue 3 + Vite 项目。
2. 配置路由和三套 Layout。
3. 封装 Axios 请求。
4. 实现 userStore 和权限路由。
5. 实现登录、注册页面。
6. 实现首页基础页面。
7. 实现电影列表和详情页。
8. 实现评分和短评交互。
9. 实现长评列表、编辑和详情。
10. 实现资讯列表和详情。
11. 实现视频和周边页面。
12. 实现管理员后台审核页。
13. 实现电影商后台投稿页。
14. 实现 AI 悬浮助手。

## 15. 前端交付检查清单

- 路由路径与文档一致。
- 页面文件命名规范。
- API 请求文件按模块拆分。
- 所有请求通过统一 request 实例。
- Token 能自动携带。
- 401 和 403 有统一处理。
- 表单必填校验完整。
- 列表页有分页。
- loading、空状态、错误状态完整。
- 审核状态文案统一。
- 前台公开页面只展示 ONLINE 内容。
- 后台页面按角色和权限显示。
